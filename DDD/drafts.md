# 领域分析收集模式
1. 选择业务案例
2. 现有模型诊断：好的价值点、不好的设计点分析
3. 改进DDD分析模型，反馈好的模式、不好的模式。
4. 适当抽象，形成适合培训交流的 DDD 案例

# 订单系统
1. 识别下游客户。
    - POS
    - APP
    - 外卖
    - 财务 这是一个非常关键的外部客户，订单系统内部的变更应该具备有不影响下游系统的能力。
2. 核心不变量设计
3. 性能设计
    - 长事务优化
4. 质量监控设计
    - 数据质量
    - 系统健壮性
    - 关注异常流程（支付一致性）
5. 日志溯源设计。
6. 原子化服务设计。
7. 抽象、一致性设计
    - 多种订单类型
    - 多种支付方式
    - 多个订单渠道

# 订购系统
1. 识别上游、下游客户。
2. 实体抽象、一致性
    - 多种采购模式
    - 多种物流模式
    - 多种结算模式
3. 如何应对流程复杂性
    - 原子操作
    - 统一状态图
4. 数据质量保障
5. 质量监控设计
6. 策略识别，将策略、流程、服务进行隔离，实现更好的动态适配能力
