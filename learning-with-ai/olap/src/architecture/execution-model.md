# 执行模型

> 待完善：本章节将对比不同 OLAP 引擎的执行模型。

## 概述

执行模型是查询引擎的核心架构之一，决定了数据如何在算子之间流动和处理。

## 主要执行模型

### 火山模型（Volcano Model）
- 也称为迭代器模型或拉取模型
- 每次调用返回一行数据
- 详见：[火山模型](./volcano-model.md)

### 向量化执行（Vectorized Execution）
- 每次处理一批数据（向量）
- 提高 CPU 缓存利用率
- 详见：[向量化执行](./vectorization.md)

### 编译执行（Compilation）
- 将查询编译为机器码
- 消除解释开销
- 详见：[编译执行](./compilation.md)
