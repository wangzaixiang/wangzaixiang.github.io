# OLAP 引擎技术对比分析

本项目聚焦于主流 OLAP 引擎的技术对比分析，重点关注架构设计和性能特点。

## 研究对象

当前聚焦于以下六款 OLAP 引擎：

### 嵌入式/进程内引擎
1. **DuckDB** - 嵌入式分析型数据库
2. **DataFusion** - Rust 实现的查询执行引擎
3. **Polars** - 高性能 DataFrame 库

### 分布式/执行引擎
4. **Spark SQL** - 分布式 SQL 引擎
5. **Trino** - 分布式查询引擎
6. **Velox** - 统一的执行引擎

## 对比维度

### 架构设计
- 执行模型（火山模型 vs 向量化 vs 编译执行）
- 查询优化器设计
- 存储格式和内存管理
- 并行化策略
- 模块化设计

### 性能特点
- 向量化执行
- SIMD 优化
- 多线程并行
- 内存管理
- 查询性能基准

## 学习方式

1. **问答记录**：将问答内容记录到 `qa/` 目录
2. **知识整理**：定期将问答整理到 mdBook 中
3. **数学建模**：从数学角度理解计算过程
4. **图表可视化**：使用 mermaid 绘制架构图

## 构建命令

```bash
# 构建 mdBook
mdbook build . -d $(pwd)/../../mysite/static/olap --open

# 开发模式
mdbook serve -d $(pwd)/../../mysite/static/olap --open .
```
